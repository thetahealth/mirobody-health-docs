---
title: "数据"
description: "POST /v1/data、GET /v1/data —— 写入并读取结构化健康记录。"
---

把结构化记录直接写入某个数据主体(Subject)的存储，并读取回来。对话智能体用其内置工具读取的正是这同一份数据。

## 写入记录

```http
POST /v1/data
Authorization: Bearer mb_live_*
Content-Type:  application/json
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `records` | array | 每条记录 `{indicator, value, unit, time}`。分批插入（每批 1000 条）。 |
| `user` | string | 记录所属的数据主体(Subject)（多租户隔离键）。 |
| `retention` | string | **必填 —— 没有默认值。** 取值为 `permanent` / `1h` / `2h` / `6h` / `1d` / `session` 之一（`persistent` 作为 `permanent` 的同义词也可接受）。小时粒度会在写入后约这么多小时自动删除（上限仍硬性限制为 ≤ 24 小时）。省略它会返回 `HTTP 400`（`code: invalid_retention`）。**`retention=none` 在此处同样会被拒绝并返回 `400`** —— 写入存储却要求不存储是自相矛盾的；用完即弃的数据请改为在 [`POST /v1/chat/completions`](/zh/api-reference/chat#内联-health_context) 上以 `health_context` + `retention=none` 内联附带。仅当该值**缺失**时才会被拒绝 —— 未知或超范围的值目前会被接受而不做校验。 |
| `session_id` | string | 用于限定 `retention=session` 记录的范围。**目前不强制** —— 不带它也会被接受，但未附带 `session_id` 写入的记录无法被会话删除操作定位到。 |

```bash
curl https://test-mirobody-api.thetahealth.ai/v1/data \
  -H "Authorization: Bearer $MIROBODY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
        "user": "alice",
        "retention": "permanent",
        "records": [
          {"indicator": "fasting_glucose", "value": 5.4, "unit": "mmol/L", "time": "2026-06-16T07:30:00Z"},
          {"indicator": "fasting_glucose", "value": 5.1, "unit": "mmol/L", "time": "2026-06-17T07:25:00Z"}
        ]
      }'
```

响应：

```json
{ "status": "ok", "ingested": 2, "subject": "alice" }
```

## 读取记录

```http
GET /v1/data?indicator=fasting_glucose&limit=100
Authorization: Bearer mb_live_*
```

| 查询参数 | 说明 |
| --- | --- |
| `user` | 要读取的数据主体(Subject)。 |
| `indicator` | 可选，按指标名称过滤。 |
| `limit` | `1`–`1000`。 |
| `offset` | 分页偏移量，默认 `0`。在返回 `limit` 条之前先跳过这么多条记录。 |

读取前会先清除已过期的记录，因此你拿到的总是数据主体(Subject)当前有效的数据。

```python Python
import requests

r = requests.get(
    "https://test-mirobody-api.thetahealth.ai/v1/data",
    headers={"Authorization": f"Bearer {MIROBODY_API_KEY}"},
    params={"user": "alice", "indicator": "fasting_glucose", "limit": 100, "offset": 0},
)
print(r.json())
```

响应是一个 OpenAI 风格的列表。记录位于 **`data`** 下，且每条记录的 `value` 是一个**格式化字符串**（例如 `"5.4 mmol/L"`），而非数字：

```json
{
  "object": "list",
  "data": [
    {
      "indicator": "fasting_glucose",
      "value": "5.4 mmol/L",
      "parsed_value": null,
      "parsed_unit": null,
      "time": "2026-06-16T07:30:00+00",
      "source": "api",
      "comment": ""
    }
  ],
  "has_more": false,
  "subject": "alice"
}
```

| 字段 | 说明 |
| --- | --- |
| `data` | 记录数组（最多 `limit` 条）。 |
| `has_more` | 当本页之后还有更多记录时为 `true` —— 递增 `offset` 以获取它们。 |
| `subject` | 记录所属的数据主体(Subject)（由 `user` 解析得到）。 |

<Note>
该响应目前**不**返回 `total`、`limit` 或 `offset` 字段 —— 请通过递增 `offset` 查询参数分页，并一直读取到 `has_more` 为 `false`。
</Note>

## 删除数据

<Note>
`/v1` 中**尚不提供**显式删除端点 —— `DELETE /v1/data`、`DELETE /v1/files/{file_key}` 和 `DELETE /v1/subjects/{user}` 在当前集群上均未实现。如今，记录通过 **[留存](/zh/api-reference/overview#数据留存)过期**移除：有时限的粒度（`1h`–`1d`）会自动删除，`session` 数据上限为 24 小时。如需完整的数据擦除请求，请联系 [Mirobody Support](mailto:developer@thetahealth.ai) —— 参见[合规](/zh/api-reference/compliance)。
</Note>

<Note>
对于完全不应持久化的一次性数据，请将其作为 `health_context` 内联附加到对话请求中、并设 `retention="none"`，而不要写到这里 —— 详见[对话 → 内联数据](/zh/api-reference/chat#内联-health_context)。
</Note>

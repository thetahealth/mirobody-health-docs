---
title: "选择你的 API"
description: "Answers API vs Agent API —— 一分钟选对接口。"
---

Mirobody 通过**两个接口**暴露同一个 grounded 健康智能体，二者都基于 Subject 真实、标准化的健康数据作答，并附带可追溯的工具轨迹。

<CardGroup cols={2}>
  <Card title="Answers API" icon="comment-dots" href="/zh/api-reference/chat">
    `POST /v1/chat/completions` —— 封闭、一发一答、有证据支撑。无客户端工具、无状态。可直接替换 OpenAI 调用。
  </Card>
  <Card title="Agent API" icon="robot" href="/zh/api-reference/responses">
    `POST /v1/responses` —— 兼容 OpenAI Responses。多轮状态、自带 function 工具、`response.*` 流式。
  </Card>
</CardGroup>

Answers 用的是更早的 Chat Completions 协议，刻意保持封闭 —— 无客户端工具、不存状态，只有 `chat.completion.chunk` 流式 —— 这让它可以直接嵌进现有的 OpenAI 集成，或者[包成一个工具](/zh/api-reference/use-as-a-tool)塞进更大的 agent 里。Agent 用的是更新的 Responses 协议：跨轮次保持状态（`store`、`previous_response_id`、`session_id`），接受你自己的 function 工具并完整交接 `function_call`，流式返回 `response.*` 事件，`openai-agents` SDK 开箱即用只需改 `base_url`。

除此之外两者完全一样：同一套服务端对[标准化数据](/zh/api-reference/data)的依据、同样的 `mirobody-flash` / `mirobody-expert` 模型、同样的 Subject 租户隔离、同样的用量口径。

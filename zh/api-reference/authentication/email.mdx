---
title: "邮箱码登录"
description: "/email/login + /email/verify 两步流程，回传一个长期 JWT access_token"
---

国内主要走「邮箱 + 6 位验证码」两步登录，回传一个长期 JWT。也支持 [OAuth 2.0](/zh/api-reference/authentication/oauth)。

## 1. 申请验证码

```http
POST /email/login
Content-Type: application/json
```

```json
{ "email": "user@example.com" }
```

响应：

```json
{ "success": true, "code": 0, "msg": "ok", "data": { "email": "user@example.com" } }
```

同一邮箱 60 秒内再次申请会被限流（HTTP 429）。

## 2. 校验并换取 token

```http
POST /email/verify
Content-Type: application/json
```

```json
{ "email": "user@example.com", "code": "000000" }
```

响应（节选）：

```json
{
  "success": true, "code": 0, "msg": "ok",
  "data": {
    "access_token":  "eyJhbGci...",
    "token_type":    "Bearer",
    "expires_in":    2592000,
    "refresh_token": "eyJhbGci...",
    "webauthn_registered": false
  }
}
```

- `access_token` 默认 **30 天**有效，到期前应提前续期，详见 [Token 续期](/zh/api-reference/authentication/refresh)。
- `refresh_token` 默认 **60 天**有效。
- JWT `sub` 字段即数字 `user_id`。

## 3. 在请求中携带 token

```text
Authorization: Bearer <access_token>
```

WebSocket（浏览器无法设置 Header）：

```text
?token=<access_token>
```

## 4. 注销 / 改昵称

```http
POST /user/del              # 注销当前账号
POST /user/update_name      # body: { "name": "新昵称" }
```

## 错误码

| HTTP | 业务 code | 含义 |
| --- | --- | --- |
| 200 | -1 | 邮件服务未配置 |
| 200 | -2 | 验证码错误 / 邮箱校验失败 |
| 200 | -3 | 内部异常 |
| 429 | - | 限流 |

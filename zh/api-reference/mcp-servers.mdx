## 即将到来的能力

对话智能体内置了针对 Subject 健康数据的工具（见 [Function calling → 内置服务端工具](/zh/api-reference/function-calling#内置服务端工具)）。MCP 接入将允许你用**自己的远程 MCP server** 扩展它：平台负责连接、列出你服务器上的工具，并让智能体在一轮中与内置工具并肩调用它们。

<Note>
这**不是**"Mirobody 作为 MCP server"。方向相反：你把外部 MCP 工具带*进*智能体的数据收集过程。调用由平台代理。
</Note>

## 规划中的形状（可能变化）

在 **Agent API** 上，使用 OpenAI Responses 的 `tools` 形状：

```jsonc
// POST /v1/responses —— 路线图，尚未上线
{
  "model": "mirobody-expert",
  "input": "Cross-check my labs against our formulary.",
  "user": "alice",
  "tools": [
    {
      "type": "mcp",
      "server_label": "formulary",
      "server_url": "https://tools.example.com/mcp",
      "authorization": "Bearer ..."
    }
  ]
}
```

你的 MCP 工具调用将通过与其他工具相同的轨迹通道呈现；安全姿态（允许工具过滤、审批流程）将在发布时一并成文。

## 今天的替代方案

MCP 接入能做到的一切，现在都可以用[**客户端 function 工具**](/zh/api-reference/function-calling)实现：声明工具、在你的进程中执行（包括转发给你自己的 MCP server），再用 `function_call_output` 续跑 —— openai-agents SDK 会把整个闭环自动化。

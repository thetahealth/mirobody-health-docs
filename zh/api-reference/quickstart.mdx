Mirobody 三步走：**① 接入数据 → ② 数据以标准化形态返回 → ③ 让 AI 使用它。** 一切兼容 OpenAI —— 你需要一个 `mb_live_*` 密钥和约 10 分钟。

<Note>
账户管理 —— **API 密钥、用量与在线 Playground** —— 位于开发者控制台 [**test-platform.mirobody.ai**](https://test-platform.mirobody.ai/)。邮箱验证码登录，打开 **API Keys** 创建一个（密钥只显示**一次**）：

```bash
export MIROBODY_API_KEY="mb_live_..."
```
</Note>

<BaseUrl />

<Steps>
  <Step title="① 接入你的数据">
    <IngestMentalModel />

    用 [`POST /v1/data`](/zh/api-reference/data) 写入结构化读数（`retention` **必填**），或用 [`POST /v1/files`](/zh/api-reference/files) 上传化验报告 —— PDF、照片、表格均可：

    ```bash
    # 结构化记录
    curl https://test-mirobody-api.thetahealth.ai/v1/data \
      -H "Authorization: Bearer $MIROBODY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
            "user": "alice",
            "retention": "permanent",
            "records": [
              {"indicator": "fasting_glucose", "value": 5.4, "unit": "mmol/L", "time": "2026-06-16T07:30:00Z"},
              {"indicator": "FBG",             "value": 5.1, "unit": "mmol/L", "time": "2026-06-17T07:25:00Z"}
            ]
          }'

    # 或者一份文档（自动 OCR 与解析）
    curl https://test-mirobody-api.thetahealth.ai/v1/files \
      -H "Authorization: Bearer $MIROBODY_API_KEY" \
      -F "user=alice" -F "file=@lab_report.pdf"
    ```

    `user` 是[租户隔离键](/zh/api-reference/overview#多租户user-字段) —— 为每个终端用户传入其稳定 id，数据绝不混淆。
  </Step>

  <Step title="② 数据以标准化形态返回">
    注意上面的两种写法 —— `fasting_glucose` 和 `FBG`。把数据读回来，两行都带有**同一个 LOINC 编码**、规范名称、UCUM 解析后的值以及 FHIR 镜像 id：

    ```bash
    curl "https://test-mirobody-api.thetahealth.ai/v1/data?user=alice&limit=10" \
      -H "Authorization: Bearer $MIROBODY_API_KEY"
    ```

    ```json
    {
      "object": "list",
      "data": [
        {
          "indicator": "FBG",
          "value": "5.1 mmol/L",
          "parsed_value": "5.1",
          "parsed_unit": "mmol/L",
          "loinc_code": "1558-6",
          "canonical_name": "Fasting glucose [Moles/volume] in Serum or Plasma",
          "fhir_resource_id": "8f3c9a1e-...",
          "time": "2026-06-17T07:25:00+00",
          "source": "api",
          "comment": ""
        }
      ],
      "has_more": true,
      "subject": "alice"
    }
    ```

    这就是[标准化管线](/zh/api-reference/standardization) —— 确定性的名称→LOINC（不让 LLM 猜码）+ UCUM + FHIR，作用于**每一次**写入。想先在文档上预览、不存任何东西，试试 [`POST /v1/extract`](/zh/api-reference/extract)。
  </Step>

  <Step title="③ 让 AI 在其上工作">
    一条有依据的回答只要三行 —— 智能体读的是标准化序列，所以问 "glucose" 能找到写成 "FBG" 的行：

    ```python Answers API（3 行）
    from openai import OpenAI
    client = OpenAI(api_key="mb_live_...", base_url="https://test-mirobody-api.thetahealth.ai/v1")
    print(client.chat.completions.create(model="mirobody-flash", user="alice",
        messages=[{"role": "user", "content": "How is my fasting glucose trending?"}]
    ).choices[0].message.content)
    ```

    而一个真正的 agent —— 带**你自己的工具** —— 只要十行。[openai-agents SDK](https://github.com/openai/openai-agents-python) 只需改 base URL 即可对接 [Agent API](/zh/api-reference/responses)：

    ```python openai-agents SDK（10 行）
    from agents import Agent, Runner, function_tool, set_default_openai_client, set_tracing_disabled
    from openai import AsyncOpenAI

    set_default_openai_client(AsyncOpenAI(
        base_url="https://test-mirobody-api.thetahealth.ai/v1", api_key="mb_live_..."))
    set_tracing_disabled(True)

    @function_tool
    def book_appointment(date: str) -> str:
        """Book a clinic appointment for the end user (ISO date)."""
        return f"Booked: {date} 09:30, Dr. Chen"

    agent = Agent(name="Health assistant", model="mirobody-flash", tools=[book_appointment])
    print(Runner.run_sync(agent, "Check my recent glucose; book a follow-up if it's trending up.").final_output)
    ```

    Mirobody 的内置工具读取 alice 的真实数据；当模型决定预约时，交接给**你的**函数。底层协议见 [Function calling](/zh/api-reference/function-calling)。
  </Step>
</Steps>

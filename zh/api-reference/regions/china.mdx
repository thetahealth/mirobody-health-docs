中国区生产环境正在为需要中国区域部署的工作负载做上线准备，其公开生产域名目前**尚不可用**。

## 规划域名

| 界面 | 规划地址 | 状态 |
| --- | --- | --- |
| API（`base_url`） | `https://api.mirobody.cn/v1` | 等待 DNS 与生产开通 |
| 开发者控制台 | `https://platform.mirobody.cn/` | 等待公开开通 |
| 网页应用 | `https://chat.mirobody.cn/` | 等待公开开通 |

在 Mirobody 明确确认已为你的账户开通前，请勿将生产集成配置到这些域名。

## API 契约

中国区部署计划提供与全球集群一致的稳定 `/v1` 资源：Models、Answers、Agent、Data、Extract、Files、Sessions 与 Subjects。模型提供方和定价可能因区域不同，因此客户端必须从实际调用的集群读取 [`GET /v1/models`](/zh/api-reference/models)。

## 使用全球 API

全球 API 目前可通过 `https://api.mirobody.ai/v1` 使用。从中国调用它会把请求数据发送到全球环境；仅在该处理位置符合你的应用与用户要求时使用。

如需了解中国区接入与当前可用性，请联系 [Mirobody Support](mailto:developer@thetahealth.ai)。发送受监管数据前应先审查[隐私与合规](/zh/api-reference/compliance)中的控制。

title: "文件"
description: "POST /v1/files、GET /v1/files、DELETE /v1/files/{file_key} —— 上传并解析报告、列出、获取解析后的文本、删除。"
---

上传一份健康文档，Mirobody 会将其解析进数据主体(Subject)的存储。PDF 和图片会经过 **OCR**；Excel / CSV 会按结构解析为健康记录。对话智能体随后会在作答时自动读取已上传的文件。

## 上传并解析

```http
POST /v1/files
Authorization: Bearer mb_live_*
Content-Type:  multipart/form-data
```

| 字段 | 说明 |
| --- | --- |
| `file` | 文档（PDF / 图片 / Excel / CSV）。 |
| `user` | 文件所属的数据主体(Subject)。**可选** —— 省略时会回退到你账户的 **`default`** 数据主体(Subject)，因此为实现按终端用户隔离，请始终传入它。 |
| `retention` | **此处可选**（与 `POST /v1/data` 不同，那里为必填）。取值与[数据留存](/zh/api-reference/overview#数据留存)相同。 |
| `session_id` | 可选；将 `retention=session` 的上传限定到某个[会话](/zh/api-reference/sessions)。 |

```bash
curl https://test-mirobody-api.thetahealth.ai/v1/files \
  -H "Authorization: Bearer $MIROBODY_API_KEY" \
  -F "user=alice" \
  -F "file=@checkup_2026.pdf"
```

响应：

```json
{
  "object": "file",
  "id": "AUmIn_R-Ddg.../b/bM--fG29X....csv",
  "filename": "checkup_2026.pdf",
  "bytes": 20544,
  "status": "processed",
  "created_at": 1782924296,
  "subject": "alice"
}
```

文件的标识符是 **`id`**（一个含斜杠的键）—— 在下方的获取与列出调用中将其用作 `{file_key}`。上传响应上的 `created_at` 是 **epoch 秒**。上传响应**不**包含指标计数或摘要。

<Note>
图片和 PDF 会被 OCR 为智能体可读的文本；结构化指标抽取是异步、尽力而为的。如需确定的结构化数据，请使用 [`POST /v1/data`](/zh/api-reference/data) 或上传 CSV / Excel 文件。
</Note>

## 列出文件

```http
GET /v1/files?user=alice
Authorization: Bearer mb_live_*
```

```json
{
  "object": "list",
  "data": [
    {
      "object": "file",
      "id": "AUmIn_R-Ddg.../b/bM--fG29X....csv",
      "file_key": "AUmIn_R-Ddg.../b/bM--fG29X....csv",
      "filename": "checkup_2026.pdf",
      "file_type": "application/pdf",
      "bytes": 20544,
      "created_at": 1782924296
    }
  ],
  "subject": "alice"
}
```

`data` 存放文件；`subject` 回显数据主体(Subject)。每一项都同时暴露 `id` 和 `file_key`（二者值相同）。`created_at` 是 **epoch 秒**，与上传响应一致。

## 获取解析后的文本

```http
GET /v1/files/{file_key}?user=alice
Authorization: Bearer mb_live_*
```

返回单个文件解析后的文本 —— 与智能体所读取的内容一致：

```json
{
  "object": "file",
  "id": "AUmIn_R-Ddg.../b/bM--fG29X....csv",
  "filename": "checkup_2026.pdf",
  "extracted_text": "date,indicator,value\n2026-06-16,fasting_glucose,5.4\n",
  "abstract": "",
  "subject": "alice"
}
```

`abstract` 可能为空（并非每种文件类型都会生成它）。

## 删除文件

```http
DELETE /v1/files/{file_key}?user=alice
Authorization: Bearer mb_live_*
```

立即将该文件从 API 表面移除（若不是该 Subject 的文件或已删除则返回 `404`）：

```json
{ "object": "file", "id": "AUmIn_R-Ddg.../b/bM--fG29X....csv", "deleted": true, "subject": "alice" }
```

有时限的上传也会通过 **[留存](/zh/api-reference/overview#数据留存)过期**自动移除。如需一次性擦除某个 Subject 的全部数据，请使用 `DELETE /v1/subjects/{user}` —— 参见[合规](/zh/api-reference/compliance)。

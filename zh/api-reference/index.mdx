**Mirobody API** 是**托管版**的 Mirobody。它就是同一套[开源引擎](/zh/index)—— 由我们替你运行，并封装成一套**兼容 OpenAI 的 `/v1` API**，附带 API 密钥、用量计量和开发者控制台。把任意 OpenAI SDK 指向 base URL、传入一个 `mb_live_*` 密钥,Agent 就会基于每位终端用户**真实、结构化的健康数据**作答,并给出可溯源的引用。

<Note>
**构建在开源之上。** 本平台是在[开源 Mirobody 引擎](/zh/index)之上的一层**定制化托管服务**—— 同一套内核,只是免去了基础设施。你无需构建、托管或运维任何东西:拿到密钥、调用 `/v1` 即可。托管 Agent 能做的一切,开源引擎同样能做 —— API 只是让它开箱即用。
</Note>

## 使用 Mirobody 的两种方式

选择适合你的路径 —— 底层引擎是同一套。

## 为什么用托管 API

- **兼容 OpenAI** —— 复用你现有的 OpenAI SDK,只需改 base URL 和密钥。
- **基于真实数据作答、可溯源** —— Agent 读取每位终端用户的健康记录,并展示所依据的来源。
- **原生多租户** —— 一个密钥,每位终端用户对应一个完全隔离的 [Subject](/zh/api-reference/overview#多租户：user-字段)。
- **零基础设施** —— 托管的密钥、存储、模型、控制台与计费,无需部署任何东西。

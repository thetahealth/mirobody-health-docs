**Mirobody API** 把杂乱的健康数据变成**标准化、AI 可用的记录** —— 并提供两个兼容 OpenAI 的接口让 agent 在其上工作。它是[开源 Mirobody 引擎](/zh/index)之上的托管层：我们托管存储、模型与密钥；你只需把 OpenAI SDK 指向 base URL 并带上 `mb_live_*` 密钥。

## 闭环：上传 → 标准化 → AI 使用

<Steps>
  <Step title="上传任何东西">
    结构化读数走 [`POST /v1/data`](/zh/api-reference/data)，化验 PDF / 照片 / 表格走 [`POST /v1/files`](/zh/api-reference/files)（含 OCR）—— 按终端用户完全隔离的 [Subject](/zh/api-reference/overview#多租户user-字段)。
  </Step>
  <Step title="每次写入都会标准化">
    名称确定性解析到 **LOINC**（不让 LLM 猜码），值归一到 **UCUM** 单位，每条读数都有 **FHIR** 镜像。`"血糖(空腹)"`、`"FBG"`、`"Glucose, fasting"` 归成一条序列。见[标准化](/zh/api-reference/standardization)。
  </Step>
  <Step title="让 AI 在其上工作">
    向 [Answers API](/zh/api-reference/chat) 要一条有依据、带证据的回答 —— 或在 [Agent API](/zh/api-reference/responses) 上构建完整 agent：你自己的工具、存储的对话，openai-agents SDK 开箱即用。
  </Step>
</Steps>

## 两个 API 接口，一套引擎

<CardGroup cols={2}>
  <Card title="Agent API —— 推荐" icon="robot" href="/zh/api-reference/responses">
    `POST /v1/responses`（兼容 OpenAI Responses）。你的 function 工具、`previous_response_id` / `session_id` 状态、`response.*` 流式。openai-agents SDK 只需改 base URL。
  </Card>
  <Card title="Answers API" icon="comment-dots" href="/zh/api-reference/chat">
    `POST /v1/chat/completions`。封闭的 grounded completion —— 一个问题、一条有证据的回答。任何 OpenAI SDK 直接替换，也非常适合[作为你自己 agent 里的一个工具](/zh/api-reference/use-as-a-tool)。
  </Card>
</CardGroup>

拿不准？[一分钟选对 API](/zh/api-reference/choose-your-api)。

## 为什么选 Mirobody

- **有依据，而非听着合理** —— 回答来自 Subject 的真实记录，附 `health_records` / `citations` 证据与完整服务端工具轨迹。
- **默认标准化** —— 每次写入都过 LOINC + UCUM + FHIR；你的分析与智能体读同一份干净序列。任何文档都可用 [`POST /v1/extract`](/zh/api-reference/extract) dry-run。
- **双重 OpenAI 兼容** —— Chat Completions *与* Responses 两个协议；现有 SDK 与 agent 框架直接可用。
- **天生多租户** —— 一个 key，每个终端用户一个隔离 Subject；按 Subject 的[被遗忘权删除](/zh/api-reference/overview#端点一览)。
- **内核开放** —— 同一引擎[开源](/zh/index)；随时可以自托管。

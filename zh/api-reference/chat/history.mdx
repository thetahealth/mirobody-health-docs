## 列出会话 / 获取单会话消息

```http
GET /api/history                  # 当前用户全部会话 summary
GET /api/history?session_id=<s>   # 该会话所有消息
```

### 不传 `session_id`

返回全部 summary，按时间倒序：

```json
{ "success": true, "code": 0, "msg": "ok",
  "data": { "summaries": [
    {
      "session_id":    "...",
      "timestamp":     "2026-05-22T03:33:00+00:00",
      "summary":       "看下我最近一周的睡眠",
      "query_user_id": "504"
    }
  ] }
}
```

> **已知限制**：`/api/history` 目前不分页，用户使用时间长后可能返回上万条。
>
> **客户端建议**：
>
> - 列表页只展示最近 N 条（如 30 条），按 `timestamp` 倒序取前 N。
> - 本地缓存 + 增量更新：保存 `last_timestamp`，下次拉取后过滤新于该时间的条目即可。
> - 分页能力会在后续版本提供，届时会保持响应结构向后兼容。

### 传 `session_id`

返回该会话内的消息列表：

```json
{ "success": true, "code": 0, "msg": "ok",
  "data": { "history": [
    {
      "role":      "user",
      "content":   "...",
      "content_dict": [],
      "timestamp": "...",
      "id":        "...",
      "agent":     "App",
      "provider":  "DeepSeekFlash",
      "messageType": "text"
    }
  ] }
}
```

`role` 为 `user` 或 `assistant`；`content_dict` 携带流式过程中的工具调用步骤列表。

## 按人聚合（家庭场景）

```http
GET /api/history_by_person
```

按可见用户（自己 + 已授权家人）聚合的会话列表，适合"切换家人视角"UI。

## 删除会话

```http
POST /api/history/delete
Authorization: Bearer <access_token>
Content-Type: application/json
```

```json
{ "session_id": "..." }
```

幂等 —— 会话不存在也返回 `ok`。

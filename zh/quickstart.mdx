## 一键部署

<Steps>
  <Step title="克隆仓库">
    ```bash
    git clone https://github.com/thetahealth/mirobody.git
    cd mirobody
    ```
  </Step>

  <Step title="运行部署脚本">
    ```bash
    ./deploy.sh
    ```

    脚本会自动完成：
    - 生成 `.env`（含 `ENV=localdb` 和新鲜的 `CONFIG_ENCRYPTION_KEY`）
    - 创建默认 `config.localdb.yaml`
    - 构建 Docker 镜像
    - 启动 Postgres、Redis 和 Mirobody 后端

    <Check>
    看到后端日志 `Uvicorn running on http://0.0.0.0:18080` 即代表启动完成。
    </Check>
  </Step>

  <Step title="至少配置一个 LLM API Key">
    打开 `config.localdb.yaml`，在以下选项里**至少**填一个：

    ```yaml config.localdb.yaml
    # DeepAgent（默认 Agent）必需
    OPENROUTER_API_KEY: 'sk-or-...'

    # 可选替代项
    OPENAI_API_KEY: 'sk-...'
    GOOGLE_API_KEY: '...'
    ANTHROPIC_API_KEY: 'sk-ant-...'
    ```

    <Warning>
    **`OPENROUTER_API_KEY` 是默认 DeepAgent 必需的**。可在 <a href="https://openrouter.ai/keys">openrouter.ai/keys</a> 申请。所有密钥在 Mirobody 首次加载时会用 `CONFIG_ENCRYPTION_KEY` 自动加密。
    </Warning>

    <Info>
    Mirobody 内置三种 Agent：
    - **DeepAgent** —— 单模型工具编排，默认选择
    - **MixAgent** —— 两阶段融合（编排者 + 回复者），平衡成本与质量
    - **BaselineAgent** —— 不调工具的直连 LLM 对话

    切换方式见 [工具概览](/zh/tools/overview)。
    </Info>
  </Step>

  <Step title="（可选）配置 Providers 与 OAuth">
    需要接入可穿戴设备或启用 Google / Apple 登录时，在 `config.localdb.yaml` 中补上对应凭据：

    ```yaml
    # 可穿戴设备
    GARMIN_CLIENT_ID: ''
    GARMIN_CLIENT_SECRET: ''

    WHOOP_CLIENT_ID: ''
    WHOOP_CLIENT_SECRET: ''

    # 第三方登录
    GOOGLE_OAUTH_CLIENT_ID: ''
    GOOGLE_OAUTH_CLIENT_SECRET: ''
    ```

    完整参考：[配置](/zh/configuration)。
  </Step>

  <Step title="打开本地 Web 应用">
    访问 <a href="http://localhost:18080">http://localhost:18080</a>，用预置 demo 账号登录：

    ```
    demo1@mirobody.ai
    777777
    ```

    其他预置账号（`demo2@mirobody.ai`、`demo3@mirobody.ai`）默认同密码 —— 详见 `config.localdb.yaml` 中的 `EMAIL_PREDEFINE_CODES`。

    登录后即可：
    - **上传文件** —— 化验单、病历、照片、音频（参见 [文件处理](/zh/concepts/file-processing)）
    - **与 Agent 对话** —— 自然语言提问你的数据
    - **关联 Provider** —— Garmin、Whoop、Oura、Renpho、Apple Health 等（参见 [使用 Providers](/zh/providers/using-providers)）
  </Step>

  <Step title="（可选）使用官方托管客户端">
    想要更精致的前端？直接访问 [https://mirobody.ai](https://mirobody.ai) —— Theta 官方托管的 Web 客户端，连接你本机的后端服务。

    <Tip>
    数据流仍然完全在你本机：托管客户端只是 UI，会直接调用你本地的 API。在客户端设置里填上你的后端地址即可。
    </Tip>
  </Step>
</Steps>


## Docker 服务

| 服务 | 端口 | 说明 |
|------|------|------|
| `mirobody` | 18080 | FastAPI 应用、MCP 服务、图表渲染器 |
| `pg` | 5432 | 内置 pgvector 的 PostgreSQL |
| `redis` | 6379 | 缓存与会话存储 |

<Tip>
查看日志：

```bash
docker compose logs -f mirobody
docker compose logs -f pg
docker compose logs -f redis
```
</Tip>

## 常用命令

<AccordionGroup>
  <Accordion title="停止全部服务" icon="stop">
    ```bash
    docker compose down
    ```
  </Accordion>

  <Accordion title="重启" icon="rotate">
    ```bash
    docker compose restart
    ```
  </Accordion>

  <Accordion title="改完代码后重新构建" icon="hammer">
    ```bash
    docker compose down
    docker compose build --no-cache
    docker compose up -d
    ```
  </Accordion>

  <Accordion title="打开 Postgres 控制台" icon="database">
    ```bash
    docker compose exec pg psql -U mirobody -d mirobody
    ```
  </Accordion>

  <Accordion title="打开 Redis CLI" icon="terminal">
    ```bash
    docker compose exec redis redis-cli
    ```
  </Accordion>
</AccordionGroup>

## 故障排查

<AccordionGroup>
  <Accordion title="18080 端口被占用" icon="circle-exclamation">
    修改 `config.localdb.yaml` 里的 `HTTP_PORT`：

    ```yaml
    HTTP_PORT: 18081
    ```

    然后 `docker compose restart mirobody`。
  </Accordion>

  <Accordion title="Postgres 连接失败" icon="database">
    ```bash
    docker compose ps pg
    docker compose logs pg
    ```
  </Accordion>

  <Accordion title="Redis 连接失败" icon="server">
    ```bash
    docker compose exec redis redis-cli ping
    ```
    应返回 `PONG`。
  </Accordion>

  <Accordion title="./deploy.sh 权限被拒" icon="lock">
    ```bash
    chmod +x deploy.sh
    ```
  </Accordion>

  <Accordion title="DeepAgent 启动失败" icon="robot">
    多半是 `OPENROUTER_API_KEY` 缺失或无效。补上后重启服务即可。
  </Accordion>
</AccordionGroup>

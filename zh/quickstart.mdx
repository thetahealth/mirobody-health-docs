---
title: "快速开始"
description: "使用 SQLite 构建 Mirobody C++ 引擎并在本地运行。"
---

本页使用自包含的 **SQLite** 后端，无需预先部署 PostgreSQL。你将构建 C++ 引擎、配置一个 LLM 密钥、启用本地演示登录，然后验证服务。

<Note>
PostgreSQL、Windows、Docker、移动端和嵌入式构建方式见[安装指南](/zh/installation)。本页只保留一条首次运行路径。
</Note>

## 1. 克隆仓库并安装依赖

```bash
git clone https://github.com/thetahealth/mirobody.git
cd mirobody
```

<Tabs>
  <Tab title="Debian / Ubuntu / WSL">
    ```bash
    sudo apt install build-essential cmake ninja-build pkg-config \
      libwebsockets-dev libcurl4-openssl-dev libssl-dev \
      rapidjson-dev libyaml-cpp-dev libhiredis-dev libsqlite3-dev \
      libjpeg-dev libpng-dev libtiff-dev libwebp-dev
    ```
  </Tab>

  <Tab title="macOS">
    ```bash
    brew install cmake ninja pkg-config libwebsockets curl openssl@3 \
      rapidjson yaml-cpp hiredis sqlite \
      jpeg-turbo libpng libtiff webp
    ```
  </Tab>

  <Tab title="Fedora / RHEL">
    ```bash
    sudo dnf install gcc-c++ cmake ninja-build pkgconf-pkg-config \
      libwebsockets-devel libcurl-devel openssl-devel \
      rapidjson-devel yaml-cpp-devel hiredis-devel sqlite-devel \
      libjpeg-turbo-devel libpng-devel libtiff-devel libwebp-devel
    ```
  </Tab>
</Tabs>

## 2. 构建 SQLite 版本

```bash
./build.sh sqlite
```

产物位于 `build-sqlite/mirobody`。构建目录使用连字符；默认的 PostgreSQL 构建产物位于 `build/mirobody`。

## 3. 配置本地服务

```bash
cp config.example.yml config.yml
```

编辑 `config.yml`：

```yaml
# 至少配置一个托管模型：
GOOGLE_API_KEY: 'AIza...'          # 或 OPENAI_API_KEY: 'sk-...'

# 本地演示登录。任何可被外部访问的部署都不应启用。
EMAIL_PREDEFINE_CODES:
  demo1@mirobody.ai: '777777'
```

<Warning>
演示凭据是公开信息，只能用于本地服务。将 Mirobody 暴露到网络之前，请配置真实的邮箱或社交登录。
</Warning>

## 4. 运行并登录

请在仓库根目录运行二进制，以便读取 `./config.yml` 和 `res/`：

```bash
./build-sqlite/mirobody
```

打开 [http://localhost:8080](http://localhost:8080)，使用以下凭据登录：

```text
demo1@mirobody.ai
777777
```

现在可以上传健康文档、与内置智能体对话，或关联数据 Provider。

## 5. 验证服务

```bash
# 存活检查
curl http://localhost:8080/api/health

# 公开的 FHIR R4 CapabilityStatement
curl http://localhost:8080/fhir/metadata

# MCP 工具发现
curl -X POST http://localhost:8080/mcp \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list"}'
```

健康检查返回 `ok`，其余两个命令返回 JSON。

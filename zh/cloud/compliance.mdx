Mirobody Platform 在中国大陆与美国分别独立运营。两个集群**完全独立** —— 账号、文件、健康数据**不会跨境复制**，每个集群按所在司法辖区的法律设计、运营、审计。

<Note>
两个集群都按 **HIPAA 健康级隐私** 标准设计；美国侧另外接受第三方持续审计。中国集群依法遵循 PRC 隐私与数据安全法律，并主动对齐 HIPAA 设计原则。
</Note>

## 核心承诺

<AccordionGroup>
  <Accordion title="健康档案隐私化" icon="user-shield">
    遵循"最小必要"原则。健康指标 / 文件 / 日记按用户维度隔离；分析层面只接触必要字段，敏感字段静态加密存储；服务运维人员不接触明文 PHI。
  </Accordion>
  <Accordion title="中间过程不留存" icon="ghost">
    上游 LLM 的中间推理 / 工具调用 trace 不持久化；仅在用户自己的会话历史中保留可见的对话内容。LLM 提供商凭证由 Mirobody 统一管理，不对外暴露。
  </Accordion>
  <Accordion title="加密保护" icon="lock">
    传输层强制 TLS 1.2+（HTTPS / WSS）；静态存储采用行业领先标准的加密；用户敏感字段在入库前另作字段级加密。
  </Accordion>
  <Accordion title="不出售用户数据" icon="ban">
    我们**绝不**将用户健康数据用于销售、广告投放或第三方变现。
  </Accordion>
  <Accordion title="用户权利" icon="key">
    账号注销（`POST /user/del` 触发软删除）；数据导出（[健康档案](/zh/api-reference/health-indicators/query) / [日记](/zh/api-reference/journal/list) / [文件](/zh/api-reference/files/list) / [对话历史](/zh/api-reference/chat/history) 均可经 API 自取）。
  </Accordion>
</AccordionGroup>

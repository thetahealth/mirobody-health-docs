## Mirobody 是什么？

**Mirobody** 是一个开源、可自托管的**健康数据 + AI 引擎** —— 一个轻量的 C++ 内核，把你的个人健康数据接入 LLM，并可以在任何地方运行，包括完全**在设备本地**运行，让数据永远无需离开手机。

完整的架构、功能列表与技术栈，见 [**GitHub README**](https://github.com/thetahealth/mirobody/blob/main-v2/README.md)——本文档站要解决的是如何真正把它跑起来、配置好、并做二次开发。

## Theta Wellness：生产环境中的 Mirobody

[**Theta Wellness**](https://www.thetahealth.ai/) 是我们构建在 Mirobody 之上的旗舰消费应用 —— 一个个人健康智能 App，展示了 Mirobody 在生产级规模下的能力。它连接可穿戴设备、手机健康存储和 EHR 病历，并支持语音、图像、文件、文字作为输入。

[**platform.mirobody.ai**](https://platform.mirobody.ai/) 是下面这套 API 对应的托管开发者控制台 —— API 密钥、用量，以及"接入 → 标准化 → 智能体"全流程的实时 Playground。想先直接体验 Mirobody、不接入任何系统？[**chat.mirobody.ai**](https://chat.mirobody.ai/) 是 Mirobody 自己的免费网页客户端 —— 标准化你的数据、直接看到引擎工作，还支持多个 AI 模型并排对比。

我们之所以开源引擎，是为了让你能自己运行同一个私密、可在设备本地运行的健康内核 —— 为全家自托管，在手机上离线运行，或经 C API 嵌入你自己的应用。

## 下一步

<Steps>
  <Step title="开始上手">
    跟着 [快速开始](/zh/quickstart) 构建内核（`./build.sh`）并运行。
  </Step>
  <Step title="配置提供商">
    给想接入的设备配好 OAuth 凭据 —— 参见 [配置](/zh/configuration)。
  </Step>
  <Step title="探索 API">
    阅读 [API 参考](/zh/api-reference/overview) 了解 OpenAI 兼容的 `/v1` 端点。
  </Step>
  <Step title="加自定义 Provider 或工具">
    新数据源走 [Provider 集成](/zh/development/provider-integration)；或在 `res/mcp_tools/` 里加一个 C++ 工具。
  </Step>
</Steps>

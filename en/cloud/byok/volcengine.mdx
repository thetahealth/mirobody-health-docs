[**Volcengine Ark**](https://www.volcengine.com/product/ark) is ByteDance's foundation-model service. It's primarily used for the **Doubao** family, plus a curated set of third-party models (DeepSeek, Kimi, Qwen).

<Note>
**Official docs**: [Ark product page](https://www.volcengine.com/product/ark) · [Model list](https://www.volcengine.com/docs/82379/1330310) · [API Key console](https://console.volcengine.com/ark/region:ark+cn-beijing/apiKey)
</Note>

## 1. Register and get a key

<Steps>
  <Step title="Enable Ark">
    Sign in to [Volcengine console](https://console.volcengine.com/ark) and enable Ark (first-time real-name verification required for mainland China users).
  </Step>
  <Step title="Subscribe to models">
    In the [Model Plaza](https://console.volcengine.com/ark/region:ark+cn-beijing/model), click **Enable** + **Try Now** on each model you need. Most Doubao models have a free trial; some are token-metered.
  </Step>
  <Step title="Create an API Key">
    On the [API Key page](https://console.volcengine.com/ark/region:ark+cn-beijing/apiKey), create a key (shown once).
  </Step>
  <Step title="(Optional) Create an Endpoint">
    Some models are called via "online inference Endpoints" — create an endpoint in the [Endpoint list](https://console.volcengine.com/ark/region:ark+cn-beijing/endpoint) to get an `ep-xxxxxxxxxxxx` identifier. Mirobody calls via model id directly, so this step is **not required** for the standard flow.
  </Step>
</Steps>

## 2. Endpoint info

| Field | Value |
| --- | --- |
| OpenAI-compatible base URL | `https://ark.cn-beijing.volces.com/api/v3` |
| Auth | `Authorization: Bearer <ARK_API_KEY>` |
| Region | Beijing (`ark+cn-beijing`, primary) |

## 3. Available models

Ark headlines the **Doubao** family, with curated third-party models. The Mirobody `/api/chat` `provider` field routes here. See the [Ark model list](https://www.volcengine.com/docs/82379/1330310) for the live catalog.

### Third-party (subscribe via Ark first)

- `deepseek-r1` / `deepseek-v3.1` — reasoning / general chat
- `moonshot-v1-128k`, `moonshot-kimi-k2` — Kimi family
- `qwen2.5-72b-instruct` and other Qwen models

## 4. Use it in Mirobody

```bash
curl -N -X POST https://mcp.thetahealth.cn/api/chat \
  -H "Authorization: Bearer $TOKEN" \
  -H "X-LLM-Provider: volcengine" \
  -H "X-LLM-Api-Key: <your Ark API key>" \
  -H "Content-Type: application/json" \
  -d '{"question":"hi","agent":"App","provider":"豆包"}'
```

## Troubleshooting

- `Model not found` → the model isn't subscribed in your Ark account; go to [Model Plaza](https://console.volcengine.com/ark/region:ark+cn-beijing/model) and enable it.
- `AuthenticationError` → API Key invalid or revoked; regenerate.
- `RateLimitExceeded` → QPS / TPM limit on Ark; adjust quotas or request whitelisting.
- Model IDs versioned with date suffix (e.g. `260428`) — Mirobody maintains the active mapping; contact your account manager if a `provider` stops working.

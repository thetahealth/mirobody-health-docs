Mirobody's hosted API is **OpenAI-compatible**: point any OpenAI SDK at `https://mcp.thetahealth.cn/v1`, pass an `mb_live_*` key, and the agent answers from each end-user's real health data.

<Steps>
  <Step title="Create an API key">
    Sign in to the developer console at [**cdm.thetahealth.cn**](https://cdm.thetahealth.cn/) (use your own email, or a [demo account](#demo-accounts) below), then go to **API Keys → Create**. Copy the `mb_live_*` secret — it's shown **once**.

    ```bash
    export MIROBODY_API_KEY="mb_live_..."
    ```
  </Step>

  <Step title="Make your first call">
    <CodeGroup>
    ```bash curl
    curl https://mcp.thetahealth.cn/v1/chat/completions \
      -H "Authorization: Bearer $MIROBODY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
            "model": "mirobody-flash",
            "messages": [{"role": "user", "content": "How is my fasting glucose trending?"}],
            "user": "demo_diabetes"
          }'
    ```

    ```python Python (OpenAI SDK)
    from openai import OpenAI

    client = OpenAI(api_key="$MIROBODY_API_KEY", base_url="https://mcp.thetahealth.cn/v1")
    resp = client.chat.completions.create(
        model="mirobody-flash",
        messages=[{"role": "user", "content": "How is my fasting glucose trending?"}],
        user="demo_diabetes",
    )
    print(resp.choices[0].message.content)
    ```

    ```javascript Node.js (OpenAI SDK)
    import OpenAI from "openai";

    const client = new OpenAI({
      apiKey: process.env.MIROBODY_API_KEY,
      baseURL: "https://mcp.thetahealth.cn/v1",
    });
    const resp = await client.chat.completions.create({
      model: "mirobody-flash",
      messages: [{ role: "user", content: "How is my fasting glucose trending?" }],
      user: "demo_diabetes",
    });
    console.log(resp.choices[0].message.content);
    ```
    </CodeGroup>

    The `user` field isolates each end-user as a [Subject](/en/api-reference/overview#multi-tenancy-the-user-field). See the full [Chat reference](/en/api-reference/chat).
  </Step>

  <Step title="Give the Subject some data">
    A fresh Subject has no data, so the agent will say so. Write a record (or upload a report with [`POST /v1/files`](/en/api-reference/files)) and ask again:

    ```bash
    curl https://mcp.thetahealth.cn/v1/data \
      -H "Authorization: Bearer $MIROBODY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
            "user": "demo_diabetes",
            "records": [{"indicator": "fasting_glucose", "value": 5.4, "unit": "mmol/L", "time": "2026-06-16T07:30:00Z"}]
          }'
    ```

    See [Data](/en/api-reference/data) and [Files](/en/api-reference/files).
  </Step>
</Steps>

## Demo accounts

Sign in to the **console** with any of these to create a key without provisioning your own account. Code is always `000000`. Each is pre-seeded with a different clinical profile:

| Console login | Profile |
| --- | --- |
| `demo_pcp@mirobody.example` | General / primary care |
| `demo_cardio@mirobody.example` | Cardiovascular |
| `demo_anxiety@mirobody.example` | Anxiety / mental health |
| `demo_knee@mirobody.example` | Orthopedic (knee) |
| `demo_cardio_history@mirobody.example` | Cardiovascular, longitudinal history |
| `demo_diabetes@mirobody.example` | Diabetes / glucose trends |

<Warning>
These accounts are **publicly shared** for evaluation. **Do not** upload real personal health data — other evaluators may read, overwrite, or clear the contents at any time. For a private partner account, email [Mirobody Support](mailto:developer@thetahealth.ai).
</Warning>

Testing against staging? Swap the base URL to `https://test-mcp.thetahealth.cn/v1` and the console to [test-cdm.thetahealth.cn](https://test-cdm.thetahealth.cn/).

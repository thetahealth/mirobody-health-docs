Mirobody is **global-first**, with **no cross-border data flow** between clusters. Every code sample in these docs targets the live **global test** cluster; swap the `base_url` for another cluster once it's live.

| Cluster | API base URL | Console | Consumer app | Status |
| --- | --- | --- | --- | --- |
| [🌐 Global](/en/cloud/regions/global) | test `https://test-mirobody-api.thetahealth.ai/v1` · prod `mirobody-api.thetahealth.ai/v1` | [platform.mirobody.ai](https://platform.mirobody.ai/) | [chat.mirobody.ai](https://chat.mirobody.ai/) | ✅ Live (test) · prod provisioning |
| [🇨🇳 China](/en/cloud/regions/china) | `https://mirobody-api.thetahealth.cn/v1` | [cdm.thetahealth.cn](https://cdm.thetahealth.cn/) | [mirobody.thetahealth.cn](https://mirobody.thetahealth.cn/) | 🚧 Provisioning |
| [🇪🇺 European Union](/en/cloud/regions/global) | not yet open externally | — | — | 🚧 Coming soon |

Cluster is fixed at contract time; **runtime switching is not supported**. See [Compliance](/en/cloud/compliance) for details.

## One-command deploy

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/thetahealth/mirobody.git
    cd mirobody
    ```
  </Step>

  <Step title="Run the deploy script">
    ```bash
    ./deploy.sh
    ```

    This script:
    - Generates a `.env` file with `ENV=localdb` and a fresh `CONFIG_ENCRYPTION_KEY`
    - Creates a default `config.localdb.yaml`
    - Builds the Docker image
    - Starts Postgres, Redis, and the Mirobody backend

    <Check>
    When you see the backend log line `Uvicorn running on http://0.0.0.0:18080`, you're good.
    </Check>
  </Step>

  <Step title="Add at least one LLM API key">
    Open `config.localdb.yaml` and fill in **at least one** of:

    ```yaml config.localdb.yaml
    # Required for DeepAgent (the default agent)
    OPENROUTER_API_KEY: 'sk-or-...'

    # Optional alternatives
    OPENAI_API_KEY: 'sk-...'
    GOOGLE_API_KEY: '...'
    ANTHROPIC_API_KEY: 'sk-ant-...'
    ```

    <Warning>
    **`OPENROUTER_API_KEY` is required for the default DeepAgent.** Get one at <a href="https://openrouter.ai/keys">openrouter.ai/keys</a>. Keys are auto-encrypted with your `CONFIG_ENCRYPTION_KEY` the first time Mirobody loads them.
    </Warning>

    <Info>
    Mirobody includes three agents:
    - **DeepAgent** — single-model tool orchestration, the default
    - **MixAgent** — two-phase fusion (orchestrator + responder) for cost/quality balance
    - **BaseAgent** — direct LLM chat, no tools

    See [Tools Overview](/en/tools/overview) for switching agents.
    </Info>
  </Step>

  <Step title="(Optional) Configure providers and OAuth">
    To connect wearables or sign in with Google/Apple, add the corresponding credentials in `config.localdb.yaml`:

    ```yaml
    # Wearables
    GARMIN_CLIENT_ID: ''
    GARMIN_CLIENT_SECRET: ''

    WHOOP_CLIENT_ID: ''
    WHOOP_CLIENT_SECRET: ''

    # OAuth sign-in
    GOOGLE_OAUTH_CLIENT_ID: ''
    GOOGLE_OAUTH_CLIENT_SECRET: ''
    ```

    Full reference: [Configuration](/en/configuration).
  </Step>

  <Step title="Open the web app">
    Visit <a href="http://localhost:18080">http://localhost:18080</a> and sign in with a preset demo account:

    ```
    demo1@mirobody.ai
    777777
    ```

    Other preset accounts (`demo2@mirobody.ai`, `demo3@mirobody.ai`) use the same password by default — see `EMAIL_PREDEFINE_CODES` in `config.localdb.yaml`.

    Once in, you can:
    - **Upload files** — lab reports, medical records, photos, audio (see [File Processing](/en/concepts/file-processing))
    - **Chat with the agent** — ask anything about your data
    - **Link providers** — Garmin, Whoop, Oura, Renpho, Apple Health (see [Using Providers](/en/providers/using-providers))
  </Step>

  <Step title="(Optional) Use the hosted client instead">
    Prefer a polished web client over the local UI? Open [https://mirobody.ai](https://mirobody.ai) — it's our official hosted frontend that talks to your local backend over your machine.

    <Tip>
    The hosted client is convenient, but data flow stays local: it connects directly to your local API. Set your backend URL in the client's settings.
    </Tip>
  </Step>
</Steps>

## Verify the API

<Tabs>
  <Tab title="Health check">
    ```bash
    curl http://localhost:18080/health
    ```
  </Tab>

  <Tab title="List providers">
    ```bash
    curl http://localhost:18080/api/v1/pulse/providers
    ```
  </Tab>

  <Tab title="MCP tools/list">
    ```bash
    curl -X POST http://localhost:18080/mcp \
      -H "Content-Type: application/json" \
      -d '{
        "jsonrpc": "2.0",
        "id": 1,
        "method": "tools/list"
      }'
    ```

## Docker services

| Service | Port | Description |
|---------|------|-------------|
| `mirobody` | 18080 | FastAPI app, MCP server, chart renderer |
| `pg` | 5432 | PostgreSQL with pgvector |
| `redis` | 6379 | Cache and session store |

<Tip>
View logs:

```bash
docker compose logs -f mirobody
docker compose logs -f pg
docker compose logs -f redis
```
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 18080 already in use" icon="circle-exclamation">
    Change `HTTP_PORT` in `config.localdb.yaml`:

    ```yaml
    HTTP_PORT: 18081
    ```

    Then `docker compose restart mirobody`.
  </Accordion>

  <Accordion title="Postgres connection failed" icon="database">
    ```bash
    docker compose ps pg
    docker compose logs pg
    ```
  </Accordion>

  <Accordion title="Redis connection failed" icon="server">
    ```bash
    docker compose exec redis redis-cli ping
    ```
    Should return `PONG`.
  </Accordion>

  <Accordion title="Permission denied on ./deploy.sh" icon="lock">
    ```bash
    chmod +x deploy.sh
    ```
  </Accordion>

  <Accordion title="DeepAgent fails to start" icon="robot">
    `OPENROUTER_API_KEY` is missing or invalid. Fill it in `config.localdb.yaml` and restart.
  </Accordion>
</AccordionGroup>

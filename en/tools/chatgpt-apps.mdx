## Overview

ChatGPT Apps extend ChatGPT with custom functionality by talking to an external MCP server. Mirobody runs as an OAuth-enabled **HTTP Remote MCP server** out of the box — all you need is a public HTTPS URL pointed at your Mirobody instance.

<Info>
Background reading: the <a href="https://openai.com/index/introducing-apps-in-chatgpt/">official ChatGPT Apps announcement</a> and the <a href="https://modelcontextprotocol.io">Model Context Protocol spec</a>.
</Info>

## How it works

Mirobody's MCP server is a first-class HTTP service. When you set `MCP_PUBLIC_URL` in your config, Mirobody uses that as the externally-reachable URL of its `/mcp` endpoint. ChatGPT (or any remote MCP client) connects there, authenticates via OAuth, and starts calling tools.

## Prerequisites

You need a publicly-reachable HTTPS URL that forwards to your Mirobody instance. The standalone binary listens on `HTTP_HOST:HTTP_PORT` (default `0.0.0.0:8080`).

<Steps>
  <Step title="Make sure Mirobody is running">
    Local: `http://localhost:8080/mcp` should return MCP JSON. Verify with:

    ```bash
    curl -X POST http://localhost:8080/mcp \
      -H "Content-Type: application/json" \
      -d '{"jsonrpc":"2.0","id":1,"method":"tools/list"}'
    ```
  </Step>

  <Step title="Choose a public-URL strategy">
    **Option A — quick demo (tunnel from your laptop)**:

    ```bash
    # ngrok
    ngrok http 8080

    # Or cloudflared
    cloudflared tunnel --url http://localhost:8080
    ```

    Use the resulting HTTPS URL (e.g. `https://abc123.ngrok.io`).

    **Option B — production (deploy with your own domain)**: terminate TLS at a reverse proxy (Caddy / nginx / a managed load balancer) and forward to your Mirobody instance.
  </Step>

  <Step title="Set MCP_PUBLIC_URL">
    Add to your `config.yml`:

    ```yaml
    MCP_PUBLIC_URL: "https://abc123.ngrok.io/mcp"   # or your production domain + /mcp
    ```

    Restart Mirobody. Your public MCP endpoint is now `https://abc123.ngrok.io/mcp`.

    <Warning>
    Treat this URL as secret-equivalent — anyone who can reach it can attempt to OAuth into your Mirobody instance.
    </Warning>
  </Step>
</Steps>

## Creating a ChatGPT App

Once `MCP_PUBLIC_URL` is set and your tunnel/domain is reachable, register the app in ChatGPT:

<Steps>
  <Step title="Access ChatGPT Apps">
    Open ChatGPT and navigate to the Apps section. You'll see your existing apps and the option to create new ones.

    <Frame>
      <img src="/images/ChatGPT App Main.png" alt="ChatGPT Apps main interface showing available apps" />
    </Frame>
  </Step>

  <Step title="Enable Developer Mode">
    Click on "Create your own" or access Developer Mode to start building a custom app. This will open the app creation interface where you can configure your Mirobody connection.

    <Frame>
      <img src="/images/ChatGPT App Dev Mode.png" alt="ChatGPT Developer Mode for creating custom apps" />
    </Frame>
  </Step>

  <Step title="Add your MCP endpoint">
    In the app configuration:
    
    1. **Name your app**: e.g. "Mirobody"
    2. **Add MCP endpoint**: your `MCP_PUBLIC_URL` plus `/mcp` (e.g. `https://abc123.ngrok.io/mcp`)
    3. **Description** (optional): describe what your app does
    4. **Test connection**: ChatGPT validates connectivity and discovers tools

    <Frame>
      <img src="/images/ChatGPT App Add New App.png" alt="Adding a new ChatGPT app with MCP endpoint configuration" />
    </Frame>

    <Tip>
    ChatGPT auto-discovers every tool registered in your Mirobody instance — the [nine built-in tools](/en/tools/built-in) plus any C++ tool you added in `res/mcp_tools/` and rebuilt.
    </Tip>
  </Step>

  <Step title="Authenticate via OAuth">
    On first call ChatGPT redirects you to Mirobody's built-in OAuth login. Sign in once and the token is reused for the rest of the ChatGPT session.
  </Step>

  <Step title="Start using your app">
    The connected ChatGPT session can now:
    - Read the user's health records (FHIR `Observation`s) via `family_health`
    - List and read the user's uploaded files via `list_files` / `read_file`
    - Store and recall long-term memory via `remember` / `recall_memory`
    - Call any custom tool you added in `res/mcp_tools/` and rebuilt

    <Check>
    Your ChatGPT App is now wired to Mirobody.
    </Check>
  </Step>
</Steps>

## Security Considerations

<Info>
**OAuth is built in.** Anyone calling the MCP endpoint must complete Mirobody's OAuth flow first, so even a leaked URL can't be used without a valid session.
</Info>

<Warning>
**Additional hardening:**

- Treat `MCP_PUBLIC_URL` as a secret; do not share it publicly
- Put the endpoint behind rate-limiting (Mirobody's `src/server/` middleware, or your reverse proxy)
- Monitor access logs for unusual patterns
- Only expose the tunnel when you're actively using it
- For production: TLS at a managed proxy, IP allowlist if possible, observability on `/mcp`
- OAuth tokens expire — users will re-authenticate periodically
</Warning>

## Troubleshooting

### Connection failed

1. Verify your tunnel / domain is reachable: `curl -I https://<your-domain>/mcp`
2. Confirm Mirobody is bound to `0.0.0.0:8080` (or your configured `HTTP_PORT`)
3. Confirm `MCP_PUBLIC_URL` matches the URL you're giving ChatGPT
4. Restart Mirobody after editing the config — it's only re-read on startup

### Tools not appearing

1. Hit `/mcp` directly with `tools/list` and confirm tools are returned
2. Check Mirobody logs for tool-loading errors at startup
3. If you added a C++ tool, confirm the file is under `res/mcp_tools/` and rebuilt with `./build.sh` (see [Adding Tools](/en/tools/adding-tools))

## Endpoint

```http
GET /v1/models
Authorization: Bearer mb_live_*
```

Lists the capability tiers you can pass as `model` to the [Answers API](/en/api-reference/chat) and the [Agent API](/en/api-reference/responses). OpenAI-compatible shape.

## Tiers

| `model` | Tier | Use case |
| --- | --- | --- |
| `mirobody-flash` | Default — fast, low cost | Everyday health Q&A, high concurrency |
| `mirobody-expert` | Deeper reasoning | Complex report interpretation, multi-step analysis |

Billing is metered per **actual token usage** — the console's [Usage](/en/api-reference/console/usage) page is the source of truth for consumption and cost. Self-hosting (SDK / open source) is free software; you pay only for your own infrastructure and LLM keys.

## Example

<CodeGroup>
```bash curl
curl https://mirobody-api.thetahealth.ai/v1/models \
  -H "Authorization: Bearer $MIROBODY_API_KEY"
```

```python Python (OpenAI SDK)
import os

from openai import OpenAI

client = OpenAI(api_key=os.environ["MIROBODY_API_KEY"], base_url="https://mirobody-api.thetahealth.ai/v1")
for m in client.models.list().data:
    print(m.id)
```
</CodeGroup>

## Response

An OpenAI-style `{ "object": "list", "data": [ … ] }`. For your `model` value, always pass a stable tier — `mirobody-flash` (default) or `mirobody-expert`.

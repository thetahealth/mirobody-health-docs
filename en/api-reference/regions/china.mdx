The China cluster keeps 100% of data within mainland China, with no cross-border replication.

<Note>
The China-region web app and developer console are live. Their domains mirror the global ones with `.cn` in place of `.ai` — `chat.mirobody.cn` (web app), `platform.mirobody.cn` (developer console). The China **API** cluster (`api.mirobody.cn`) is coming soon — until then, create your `mb_live_*` keys on the [global console](https://platform.mirobody.ai/) and call the global API.
</Note>

## Hosts

| Surface | URL |
| --- | --- |
| **API** (`base_url`) | `https://api.mirobody.cn/v1` (coming soon) |
| **Developer console** | [platform.mirobody.cn](https://platform.mirobody.cn/) |
| **Web app** | [chat.mirobody.cn](https://chat.mirobody.cn/) |

The console is where you create `mb_live_*` keys, view usage, and try the Playground. The Mirobody web app is the flagship product — the same engine behind this API.

## Models

Managed by Mirobody — no LLM key of your own needed: `mirobody-flash` (default) / `mirobody-expert`. See [Models](/en/api-reference/models).

## Storage & residency

- Files stored in **Aliyun OSS (Hangzhou region)**.
- 100% of data stored inside mainland China; no cross-border replication.

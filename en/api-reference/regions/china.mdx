The China production environment is being prepared for workloads that must use a China-region deployment. Its public production domains are **not available yet**.

## Planned hosts

| Surface | Planned URL | Status |
| --- | --- | --- |
| API (`base_url`) | `https://api.mirobody.cn/v1` | Pending DNS and production enablement |
| Developer console | `https://platform.mirobody.cn/` | Pending public enablement |
| Web app | `https://chat.mirobody.cn/` | Pending public enablement |

Do not configure a production integration against these hosts until Mirobody confirms availability for your account.

## API contract

The China deployment is intended to expose the same stable `/v1` resources as the global cluster: Models, Answers, Agent, Data, Extract, Files, Sessions, and Subjects. Model providers and pricing can differ by region, so clients must read [`GET /v1/models`](/en/api-reference/models) from the cluster they call.

## Using the global API

The global API remains available at `https://api.mirobody.ai/v1`. Calling it from China sends request data to the global environment; do so only if that processing location is acceptable for your application and users.

For China-region onboarding and current availability, contact [Mirobody Support](mailto:developer@thetahealth.ai). See [Privacy & Compliance](/en/api-reference/compliance) for the controls you should review before sending regulated data.

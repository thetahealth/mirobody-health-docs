Mirobody is **global-first**, with **no cross-border data flow** between clusters. Every code sample in these docs uses the global cluster's `base_url`; swap it for your contracted cluster.

| Cluster | API base URL | Console | Web app | Status |
| --- | --- | --- | --- | --- |
| [🌐 Global](/en/api-reference/regions/global) | `https://api.mirobody.ai/v1` | [platform.mirobody.ai](https://platform.mirobody.ai/) | [chat.mirobody.ai](https://chat.mirobody.ai/) | ✅ |
| [🇨🇳 China](/en/api-reference/regions/china) | `https://api.mirobody.cn/v1` (coming soon) | [platform.mirobody.cn](https://platform.mirobody.cn/) | [chat.mirobody.cn](https://chat.mirobody.cn/) | 🚧 API only |

Cluster is fixed at contract time; **runtime switching is not supported**. See [Compliance](/en/api-reference/compliance) for details.

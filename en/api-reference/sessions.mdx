## Endpoint

```http
DELETE /v1/sessions/{session_id}
Authorization: Bearer mb_live_*
```

Marks a session ended. **Caveat:** on the current cluster this returns `deleted: 0` and does **not** purge `retention=session` records written via [`POST /v1/data`](/en/api-reference/data) — those still expire on the 24h session cap. Don't rely on it as your only right-to-be-forgotten mechanism.

```bash
curl -X DELETE "https://test-mirobody-api.thetahealth.ai/v1/sessions/sess_abc123?user=alice" \
  -H "Authorization: Bearer $MIROBODY_API_KEY"
```

<Note>
Pass the same `user` you used when writing the session's data; omitting it targets your account's default Subject.
</Note>

Response:

```json
{ "status": "ok", "session_id": "sess_abc123", "deleted": 0, "subject": "alice" }
```

| Field | Description |
| --- | --- |
| `status` | `"ok"` on success. |
| `session_id` | The session that was ended. |
| `deleted` | Count of purged records. **Currently returns `0`** for records written via `POST /v1/data` — see the caveat above. |
| `subject` | The Subject the session belonged to. |

## How sessions work

A `session_id` does two things in [`/v1/chat/completions`](/en/api-reference/chat):

1. **Threads multi-turn context** across calls that share the same `session_id`.
2. **Scopes `retention=session` data** — records / files / inline `health_context` written with `retention=session` live only as long as the session.

Session data is capped at **24h** regardless. (`DELETE /v1/sessions/{id}` is intended to clear it sooner, but currently reports `deleted: 0` for `POST /v1/data` records — rely on the 24h cap.)

```python Python
client.chat.completions.create(
    model="mirobody-flash",
    messages=[{"role": "user", "content": "Here are today's readings — anything off?"}],
    user="alice",
    extra_body={
        "session_id": "sess_abc123",
        "retention": "session",
        "health_context": [{"indicator": "systolic_bp", "value": 148, "unit": "mmHg", "time": "2026-06-16T08:00:00Z"}],
    },
)
# ... later, when the conversation ends:
# DELETE /v1/sessions/sess_abc123  → marks the session ended (see caveat)
```

See [retention](/en/api-reference/overview#data-retention) for the full model.

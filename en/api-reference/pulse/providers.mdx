## Endpoint

```
GET /api/v1/pulse/providers
```

## Overview

Retrieve information about all health device providers available in the system, including their configuration status and capabilities.

## Request

<RequestExample>
```bash cURL
curl http://localhost:18080/api/v1/pulse/providers
```
</RequestExample>

## Response Fields

<ResponseField name="code" type="integer" required>
  Response code (0 = success)
</ResponseField>

<ResponseField name="msg" type="string" required>
  Response message
</ResponseField>

<ResponseField name="data" type="object" required>
  Response data containing providers array

  <Expandable title="data object">
    <ResponseField name="providers" type="array" required>
      Array of provider objects

      <Expandable title="Provider object">
        <ResponseField name="slug" type="string" required>
          Unique provider identifier (e.g., `theta_garmin`, `apple_health`)
        </ResponseField>

        <ResponseField name="name" type="string" required>
          Display name of the provider
        </ResponseField>

        <ResponseField name="platform" type="string" required>
          Platform type: `theta` (OAuth providers), `apple` (file-based), `vital` (Vital API)
        </ResponseField>

        <ResponseField name="description" type="string">
          Provider description and capabilities
        </ResponseField>

        <ResponseField name="logo" type="string">
          URL to provider logo image
        </ResponseField>

        <ResponseField name="supported" type="boolean" required>
          Whether the provider is currently supported
        </ResponseField>

        <ResponseField name="auth_type" type="string" required>
          Authentication method: `oauth1`, `oauth2`, `file_upload`, `token`
        </ResponseField>

        <ResponseField name="status" type="string" required>
          Provider status: `available`, `unavailable`, or `maintenance`
        </ResponseField>

        <ResponseField name="capabilities" type="array">
          List of data types the provider supports (e.g., `activity`, `sleep`, `heart_rate`)
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Platform Types

| Platform | Description | Examples |
|----------|-------------|----------|
| `theta` | OAuth-based wearable device providers | Garmin, Whoop, Fitbit |
| `apple` | Apple Health platform (file upload) | Apple Health Export, CDA documents |
| `vital` | Vital API integration | Various devices via Vital |

## Example Usage

<CodeGroup>
```python Python
import aiohttp

async def list_providers():
    async with aiohttp.ClientSession() as session:
        async with session.get("http://localhost:18080/api/v1/pulse/providers") as resp:
            providers = await resp.json()
            return providers["providers"]

providers = await list_providers()
for provider in providers:
    print(f"{provider['name']}: {provider['status']}")
```

```bash cURL
curl http://localhost:18080/api/v1/pulse/providers | jq '.providers[] | {name, status}'
```
</CodeGroup>

<Tip>
Use the `slug` field when linking providers or querying provider-specific data.
</Tip>

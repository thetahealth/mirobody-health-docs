## Prerequisites

## Installation Steps

<Steps>
  <Step title="Clone the repository">
    Clone the Mirobody Health repository to your local machine:

    ```bash
    git clone https://github.com/thetahealth/mirobody-health.git
    cd mirobody-health
    ```
  </Step>

  <Step title="Set up configuration files">
    Create your configuration files from the provided examples:

    ```bash
    cp .env_example .env
    cp config.example.yaml config.yaml
    ```
    
    <Tip>
    For a quick test, you can use the default configuration. For production, see the [Configuration guide](/configuration).
    </Tip>
  </Step>

  <Step title="Add API keys">
    Edit `config.yaml` to add your API keys. **OPENROUTER_API_KEY is required** for AI features:

    ```yaml config.yaml
    # AI Provider API Keys - OPENROUTER_API_KEY is REQUIRED
    OPENROUTER_API_KEY: 'your_openrouter_api_key'  # REQUIRED
    
    # Optional: Additional AI providers
    GOOGLE_API_KEY: 'your_google_api_key'
    OPENAI_API_KEY: 'your_openai_api_key'
    ```

    <Warning>
    **OPENROUTER_API_KEY is mandatory** for AI chat and agent features. Get your API key at [OpenRouter](https://openrouter.ai/).
    </Warning>
    
    <Tip>
    OpenRouter provides access to multiple LLM providers through a single API, making it the recommended choice for Mirobody Health.
    </Tip>
  </Step>

  <Step title="Deploy with Docker">
    Run the deployment script to start all services:

    ```bash
    ./deploy.sh
    ```

    This will start three services:
    - **Backend API** on port 18080
    - **PostgreSQL** database on port 5432
    - **Redis** cache on port 6379

    <Check>
    Wait for all services to start. You should see logs indicating successful startup.
    </Check>
  </Step>

  <Step title="Verify the installation">
    Open your browser and navigate to:

    ```
    http://localhost:18080
    ```

    You should see the Mirobody Health welcome page.
  </Step>

  <Step title="Log in">
    Log in with the default demo account or the accounts set up in `config.yaml`.

    ``` 
    demo@mirobody.ai
    777777
    ```

    Then you can upload files and start chatting. To set up connection to data providers, see [using providers](/providers/using-providers).
  </Step>
</Steps>

## Test the API

Once the services are running, you can test the API endpoints:

<Tabs>
  <Tab title="Health Check">
    Verify the server is running:

    ```bash
    curl http://localhost:18080/health
    ```
  </Tab>

  <Tab title="List Providers">
    Get available health providers:

    ```bash
    curl http://localhost:18080/api/v1/pulse/providers
    ```
  </Tab>

  <Tab title="MCP Interface">
    Test the MCP protocol endpoint:

    ```bash
    curl -X POST http://localhost:18080/mcp \
      -H "Content-Type: application/json" \
      -d '{
        "jsonrpc": "2.0",
        "id": 1,
        "method": "tools/list"
      }'
    ```
  </Tab>
</Tabs>

## Docker Services

The deployment includes three containerized services:

| Service | Port | Description |
|---------|------|-------------|
| backend | 18080 | FastAPI application server |
| postgres | 5432 | PostgreSQL database |
| redis | 6379 | Redis cache and session store |

<Tip>
You can view logs for any service using:

```bash
docker-compose logs -f backend
docker-compose logs -f postgres
docker-compose logs -f redis
```
</Tip>

## Common Commands

<AccordionGroup>
  <Accordion title="Stop all services" icon="stop">
    ```bash
    docker-compose down
    ```
  </Accordion>

  <Accordion title="Restart services" icon="rotate">
    ```bash
    docker-compose restart
    ```
  </Accordion>

  <Accordion title="View logs" icon="file-lines">
    ```bash
    # All services
    docker-compose logs -f
    
    # Specific service
    docker-compose logs -f backend
    ```
  </Accordion>

  <Accordion title="Rebuild containers" icon="hammer">
    ```bash
    docker-compose down
    docker-compose build --no-cache
    docker-compose up -d
    ```
  </Accordion>

  <Accordion title="Access database" icon="database">
    ```bash
    docker-compose exec postgres psql -U holistic_user -d holistic_db
    ```
  </Accordion>

  <Accordion title="Access Redis CLI" icon="terminal">
    ```bash
    docker-compose exec redis redis-cli
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port already in use" icon="circle-exclamation">
    If port 18080 is already in use, edit `config.yaml`:

    ```yaml
    HTTP_PORT: 18081  # Change to an available port
    ```

    Then restart the services.
  </Accordion>

  <Accordion title="Database connection failed" icon="database">
    Ensure PostgreSQL container is running:

    ```bash
    docker-compose ps postgres
    ```

    Check database logs:

    ```bash
    docker-compose logs postgres
    ```
  </Accordion>

  <Accordion title="Redis connection failed" icon="server">
    Verify Redis is running and accessible:

    ```bash
    docker-compose exec redis redis-cli ping
    ```

    Should return `PONG` if working correctly.
  </Accordion>

  <Accordion title="Permission denied on deploy.sh" icon="lock">
    Make the script executable:

    ```bash
    chmod +x deploy.sh
    ```
  </Accordion>
</AccordionGroup>
